# Operator Responsibilities

Requirements that are **deployment/infrastructure concerns**, not library code. These are non-normative recommendations extracted from the specification's security, privacy, and implementation consideration sections.

> **Note:** Some entries below use lowercase RFC 2119 keywords (`should`, `may`) as they appear in non-normative prose within the specification. These do **not** carry the same weight as uppercase keywords defined in RFC 2119 §1. Entries using uppercase keywords (`MUST`, `SHOULD`) are normative but fall outside the scope of library code.

**Spec:** [OpenID Federation 1.0](https://openid.net/specs/openid-federation-1_0.html) (Final, 17 February 2026)
